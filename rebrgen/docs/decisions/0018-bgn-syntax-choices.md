# brgen 定義言語の構文選択

## 日付

- 判断時期: 2023年9〜10月頃（brgen プロジェクト初期）
- 文書化: 2026-04-15

## 判断

brgen の定義言語をインデントベース・Python 風の構文にする。

## 動機

- **easy to write and read**: この言語の性質は宣言的に何かを書くのが主なので、
  可読性が高いほうがよい。
- **統一的な見た目**: Python 風にしておけばユーザー間で書き方が大体同じになる。
- 当初プロトタイプ版では `{}` ベースだったが、`{}` を書くのが地味に面倒だった。
- 個人的な理由: JIS 配列キーボードでは `{}` に Shift キーが必要だが
  `:` は不要なので少し楽。

## 具体例

```
format TCPHeader:
    src_port :u16
    dst_port :u16
    seq_number :u32
```

## これは X を意味しない

- Python そのものではない。構文が似ているだけで、セマンティクスはバイナリフォーマット定義に特化。
