# ebmgen の transform pass の構成

## 日付

2026-04-15

## 判断

ebmgen に複数の transform pass を設け、EBM 生成後に IR レベルで変換・最適化を行う。

## 動機

- EBM 生成直後の表現では、そのままコード生成するには情報が不足したり
  冗長だったりするケースがある。transform pass で補完・整理する。

## 具体例

- **CFG / bit manipulation**: 動的なビットフィールドの解析のために必要。
  CFG はビットフィールドの動的な振る舞いを解析するために導入され、
  bit manipulation パスがその結果を使って処理する。
- **IO vectorization**: 一般的なベクトル化ではない。複数の固定長フィールド群を
  ひとまとめにして、一括読み込みや境界チェックの一括適用・省略のヒントとして提供する。
- **unused node removal**: DCE（Dead Code Elimination）ではない。
  IR ノードのうち参照されていないものを除去しているだけで、
  コードの論理的な到達不能性は判定していない。

## これは X を意味しない

- 一般的なコンパイラの最適化パスとは性質が異なる。
  あくまでコード生成のための前処理・情報付加であり、
  実行速度の最適化を目的としていない。
