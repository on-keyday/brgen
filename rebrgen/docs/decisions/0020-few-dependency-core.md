# コアに外部依存を混ぜない (few dependency core)

## 日付

2026-04-15

## 判断

brgen/rebrgen のコア部分に外部ライブラリへの依存を持ち込まない。
依存するのは C++ 標準ライブラリと futils（自作ユーティリティライブラリ）のみ。

## 動機

- サプライチェーン攻撃リスクの低減。
- 個人的な美学として、コアに外部依存を混ぜたくない。

## 具体例

- src2json の AST 出力形式に JSON を採用したのは、futils に JSON ライブラリが
  既にあったため。protobuf 等を使えばより効率的な表現が可能だが、外部依存になる。
- futils 自体は brgen 以前から開発していた独立したユーティリティライブラリ
  （QUIC 実装が丸ごと入っている等、brgen 専用ではない）。

## これは X を意味しない

- JSON が最適な中間形式だという主張ではない。EBM 系統が成熟したらバイナリ形式への
  切り替えもありうるが、bootstrap できなくなるリスクがあるため当分は JSON も残す。
- ツール系やテスト系での外部依存まで禁止しているわけではない。
  あくまでコア（パーサー、IR、コードジェネレーター）の話。
