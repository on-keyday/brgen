# コードジェネレーター実装を C++ に統一した

## 日付

- 判断時期: 2024年12月頃（rebrgen プロジェクト開始時、ADR-0014 と同時期の判断）
- 文書化: 2026-04-15

## 判断

ebm2xxx 系のコードジェネレーターおよび ebmcodegen を C++ で統一する。
各ターゲット言語のジェネレーターをその言語自身で実装することはしない。

## 動機

- brgen では json2go を Go で、json2rust を Rust で書くというアプローチを取っていたが、
  実際にやってみると大した旨味がなかった。
- 一人 + AI の開発体制では、コードジェネレーター自体の実装言語が分散していても
  メリットが薄い。むしろ ebmcodegen による自動生成やツールチェーンの統一が活きる。

## これは X を意味しない

- ターゲット言語自身で書くことに技術的な問題があるわけではない。
  開発体制と実際の経験に基づく実際的な判断。
