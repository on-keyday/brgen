# rebrgen を C++23 で実装する

## 日付

- 判断時期: 2024年12月（rebrgen プロジェクト開始時点からの判断）
- 文書化: 2026-04-15

## 判断

rebrgen の実装言語として C++23 を採用する。

## 動機

- **フロントエンドとの接続**: brgen のフロントエンド（src2json）が C++ で書かれており、
  同じ言語であれば直接接続できる。実際 ebmgen には direct ast pass という
  フロントとバックがともに C++ であることを前提としたコードパスがある。
- **astlib/ast2rust の使いづらさ**: Rust 用の AST ライブラリ (ast2rust) は存在するが、
  Rc<RefCell<T>> ベースで非常に使いづらく、これで本格的な開発をするのは現実的でなかった。
- **個人的な理由**: Rust のコンパイラが中間生成物を大量に生成してストレージに優しくない点が嫌だった。

## 具体例

- ebmgen の direct ast pass: src2json の AST を直接 C++ のポインタで受け取り、
  JSON シリアライズ/デシリアライズを経由せずに EBM に変換できる。
  これはフロントとバックが同じ言語だからこそ可能。

## これは X を意味しない

- brgen のコードジェネレーターの出力先が C++ に限定されるわけではない。
  rebrgen の実装言語と、コードジェネレーターが出力するターゲット言語は別の話。
- C++ 以外の言語を否定しているわけではない。実際的な制約と好みによる選択。
