# READ_DATA / WRITE_DATA Semantics

## Overview

READ_DATA and WRITE_DATA are the core IO operations in EBM IR. They exist at two levels:
**Expression** (DSL-level explicit IO like `input.get(u8)`) and **Statement** (field encode/decode IO).

## Two Levels

### Expression Level (ExpressionKind.READ_DATA / WRITE_DATA)

Generated from `ast::IOOperation` in `ebmgen/convert/expression.cpp`.

| Field | READ_DATA | WRITE_DATA |
|---|---|---|
| target_stmt | Variable definition to read into (StatementRef) | — |
| target_expr | — | Expression to write from (ExpressionRef) |
| io_statement | Reference to inner Statement READ_DATA | Reference to inner Statement WRITE_DATA |

Code generation: Expression READ_DATA visits `target_stmt` (variable def) + `io_statement` (actual IO), returns the variable name. Expression WRITE_DATA visits `io_statement` and returns its result.

Transform: `flatten_io_expression.cpp` expands expression-level READ_DATA used as statement into `{target_stmt_def; io_statement}` block.

### Statement Level (StatementKind.READ_DATA / WRITE_DATA)

Generated from `decode_field_type()` / `encode_field_type()` in `ebmgen/convert/decode.cpp` / `encode.cpp`.

This is the core of the IO pipeline. Both use the shared `IOData` structure.

## IOData Structure

```
format IOData:
    io_ref       : StatementRef      # IO stream variable
    field        : WeakStatementRef   # Mapped FieldDecl
    target       : ExpressionRef      # Variable/expression to read into or write from
    data_type    : TypeRef            # Data type
    attribute    : IOAttribute        # Endianness, sign, peek, lowered flag, etc.
    size         : Size               # Size information
    [lowered_statement]               # Present when has_lowered_statement == 1
    [offset]                          # Present when has_offset == 1
```

Note: `target` is a write destination for READ_DATA but a read source for WRITE_DATA.
This asymmetry is implicit (determined by StatementKind context, not by type).

## IOAttribute Flags

| Flag | Meaning |
|---|---|
| endian | unspec / big / little / dynamic / native |
| sign | Signed data |
| is_peek | Peek read (don't advance stream position) |
| has_lowered_statement | LoweredIOStatement exists |
| has_offset | Offset info exists (e.g., after bit-field transform) |

## Size (SizeUnit)

| SizeUnit | Meaning | Generated when |
|---|---|---|
| BIT_FIXED | Fixed bit count | Bit fields (bit_size % 8 != 0) |
| BYTE_FIXED | Fixed byte count | Normal int/float, fixed-length byte arrays |
| ELEMENT_FIXED | Fixed element count | Fixed-length non-byte arrays |
| BIT_DYNAMIC | Dynamic bit count (expr ref) | — |
| BYTE_DYNAMIC | Dynamic byte count (expr ref) | Variable-length byte arrays |
| ELEMENT_DYNAMIC | Dynamic element count (expr ref) | Variable-length non-byte arrays |
| DYNAMIC | Indeterminate | Struct calls, constant-follow arrays |

## LoweredIOStatement

When `has_lowered_statement == 1`, the IO has a lowered representation that codegen can delegate to.

| LoweringIOType | Meaning | Generated by |
|---|---|---|
| INT_TO_BYTE_ARRAY | Int ↔ byte array with endian handling | decode/encode_multi_byte_int_with_fixed_array |
| FLOAT_TO_BYTE_ARRAY | Float → byte array | encode_float_type |
| ENUM_UNDERLYING_TO_INT | Enum → base int then IO | decode/encode_enum_type |
| ARRAY_FOR_EACH | Element-by-element loop for arrays | decode/encode_array_type |
| STRING_FOR_EACH | Byte-by-byte string literal verify/construct | decode/encode_str_literal_type |
| STRUCT_CALL | Sub-struct decode/encode function call | decode/encode_struct_type |
| BIT_FIELD_TO_BIT_SHIFT | Bit shift/mask operations | transform/bit_fields.cpp |
| VECTORIZED_IO | Grouped contiguous fixed-size IOs | transform/io_vectorized.cpp |
| MULTI_REPRESENTATION | Multiple lowering candidates | — |

## Code Generation Dispatch Flow (Default Visitor)

```
Statement_READ_DATA / Statement_WRITE_DATA
  |
  +- read/write_data_custom?      -> Full override (return pass for default)
  |
  +- lowered == VECTORIZED_IO?    -> Delegate to lowered
  |
  +- read/write_data_bytes_io_wrapper?  -> Bytes-specific language handling
  |
  +- Other lowered_statement?     -> Delegate to lowered
  |
  +- GeneratorDefaultHook         -> Language-specific fallback
```

"No lowered" arrival at this flow = primitive byte-level IO.
"Has lowered" = almost always delegated to lowered statement via recursive visit.

## Per-Type IO Patterns

| Type | size | lowered | Notes |
|---|---|---|---|
| int (1 byte) | BYTE_FIXED(1) | INT_TO_BYTE_ARRAY | u8[1] buffer |
| int (N bytes) | BYTE_FIXED(N) | INT_TO_BYTE_ARRAY | u8[N] buffer + endian |
| int (bit field) | BIT_FIXED(n) | After transform: BIT_FIELD_TO_BIT_SHIFT | — |
| float | BYTE_FIXED(N) | INT/FLOAT_TO_BYTE_ARRAY | — |
| enum | (from base type) | ENUM_UNDERLYING_TO_INT | Cast + delegate to base |
| fixed array (u8) | BYTE_FIXED(N) | ~~ARRAY_FOR_EACH~~ → none | Primitive bytes IO |
| fixed array (other) | ELEMENT_FIXED(N) | ARRAY_FOR_EACH | Element loop |
| dynamic array (u8) | BYTE_DYNAMIC(expr) | ~~ARRAY_FOR_EACH~~ → none | Primitive bytes IO |
| dynamic array (other) | ELEMENT_DYNAMIC(expr) | ARRAY_FOR_EACH | Element loop |
| string literal | BYTE_FIXED(N) | STRING_FOR_EACH | Verify/construct bytes |
| struct | DYNAMIC | STRUCT_CALL | Function call |

Note: "~~ARRAY_FOR_EACH~~ → none" for u8 arrays reflects the refactoring described below.

## Transforms (Post-Generation)

Applied after ebmgen conversion:

1. **flatten_io_expression** — Expand expression-level READ_DATA into block
2. **bit_fields** — Convert bit fields to byte read + shift/mask (BIT_FIELD_TO_BIT_SHIFT)
3. **io_vectorized** — Group contiguous fixed-size IOs into single VECTORIZED_IO

VECTORIZED_IO is an optimization hint: "these contiguous IOs could be a single memcpy".
The original individual IOs are preserved inside the lowered statement so backends can
choose between bulk transfer and individual processing.

## Bytes Array as Primitive IO (Refactoring Note)

Previously, u8 arrays were treated identically to non-byte arrays: ebmgen generated
ARRAY_FOR_EACH loops with per-element IO (creating `[1]u8` temporary buffer for each byte).
Code generators then detected bytes via `is_bytes_type()` and bypassed the lowered loop
entirely, doing bulk IO instead.

This was refactored so that u8 arrays skip ARRAY_FOR_EACH generation in ebmgen,
arriving at code generators as primitive bytes IO (size set, no lowered).
The exception is `Follow::constant` arrays (peek-until-marker), which genuinely
need element-by-element processing.
